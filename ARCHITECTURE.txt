  TOKEN LIFECYCLE:
  ================

  1. User authenticates вҶ’ Backend issues:
     вҖў Access Token  (JWT, 15 min expiry)
     вҖў Refresh Token (JWT, 7 day expiry, hashed in MongoDB)

  2. Frontend stores both tokens in localStorage

  3. Each API request sends: Authorization: Bearer <accessToken>

  4. When access token expires:
     вҖў Frontend calls POST /api/auth/refresh with refreshToken
     вҖў Backend verifies вҶ’ deletes old session вҶ’ creates new tokens (rotation)

  5. MongoDB TTL index auto-purges expired sessions every ~60 seconds
